▸ Available Unenrollment Methods
▸ Utilities
GBB firmware▾

Modify GBB flags to disable firmware restrictions and enrollment enforcement.

If prompted for "PP", press power button repeatedly.
gsctool -a -o gsctool -a -I AllowUnverifiedRo:always
gsctool -a -w disable
flashrom --wp-disable
futility gbb --flash --set --flags=0xa0b1

# Alternative method:
cd /tmp
flashrom -r -i GBB:gbb.bin
futility gbb --set --flags=0xa0b1 gbb.bin
flashrom -w -i GBB:gbb.bin -N
SAUB auto▾

Automated unenrollment script for managed devices. Requires developer mode. If not in dev mode, use DAUB instead.

Connect to internet, run this command, enter 1, restart, repeat, then exit and reboot.
curl -sSL https://raw.githubusercontent.com/CriticalHD/Myscripts/refs/heads/main/SAUB | sudo bash
DAUB recovery▾

Direct unenrollment via BadSH1MMER — a modified recovery image that automatically deprovisions the device on boot. Works on many Cr50 boards where standard methods fail. Use this if not in developer mode.

Instructions: Download BadSH1MMER for your board, flash to USB, boot in recovery mode, and follow the on-screen prompts for automatic unenrollment.
